Detailed Description

template<>
class cl_algo::GF::CombineRGB< CombineRGBConfig::FLOAT_FLOAT >

Interface class for the combineRGBChannels_Float2Float kernel.

combineRGBChannels_Float2Float performs array transposition on three, R, G, and B, images. For more details, look at the kernel's documentation.

Note
The combineRGBChannels_Float2Float kernel is available in kernels/imageSupport_kernels.cl.
This is a specialization for the case of float input and output data.
The class creates its own buffers. If you would like to provide your own buffers, call get to get references to the placeholders within the class and assign them to your buffers. You will have to do this strictly before the call to init. You can also call get (after the call to init) to get a reference to a buffer within the class and assign it to another kernel class instance further down in your task pipeline.

The following input/output OpenCL memory objects are created by a CombineRGB<CombineRGBConfig::FLOAT_FLOAT> instance:

Name Type Placement I/O Use Properties Size
H_IN_R Buffer Host I Staging CL_MEM_READ_WRITE \( width*height*sizeof\ (cl\_float)\)
H_IN_G Buffer Host I Staging CL_MEM_READ_WRITE \( width*height*sizeof\ (cl\_float)\)
H_IN_B Buffer Host I Staging CL_MEM_READ_WRITE \( width*height*sizeof\ (cl\_float)\)
H_OUT Buffer Host O Staging CL_MEM_READ_WRITE \(3*width*height*sizeof\ (cl\_float)\)
D_IN_R Buffer Device I Processing CL_MEM_READ_ONLY \( width*height*sizeof\ (cl\_float)\)
D_IN_G Buffer Device I Processing CL_MEM_READ_ONLY \( width*height*sizeof\ (cl\_float)\)
D_IN_B Buffer Device I Processing CL_MEM_READ_ONLY \( width*height*sizeof\ (cl\_float)\)
D_OUT Buffer Device O Processing CL_MEM_WRITE_ONLY \(3*width*height*sizeof\ (cl\_float)\)
